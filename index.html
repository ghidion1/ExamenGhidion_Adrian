I. Inițializare Proiect

Creare repository pe GitHub 1/6

Am creat un nou repository pe GitHub cu numele ExamenGhidion_Adrian în cadrul contului ghidion1.

Repository-ul a fost inițializat cu un fișier README și este setat ca public.

Repository creat cu succes
Clonare repository local 2/6
git clone https://github.com/ghidion1/ExamenGhidion_Adrian.git cd ExamenGhidion_Adrian

Am clonat repository-ul pe calculatorul local în directorul Desktop.

După clonare, am navigat în directorul proiectului.

Adăugare fișier README.md 3/6
echo "Biletu1 nr. X - Describrea protectului: Acest protect este realizat pentru examenul la disciplina 5.08.0.027 și are ca scop gestionarea structurii unui protect web." > README.md git add README.md git commit -m "Adaugat file README.md cu descrierea biletului"

Am creat fișierul README.md cu descrierea proiectului și numărul biletului.

Am adăugat fișierul la staging area și am făcut commit cu un mesaj descriptiv.

Notă: Mesajul commit-ului conține typo ("comnit" în loc de "commit")
Adăugare fișier .gitignore 4/6
echo "*.log" > .gitignore echo "node_modules/" >> .gitignore git add .gitignore git commit -m "Adaugat fisier .gitignore pentru loguri si node_modules"

Am creat fișierul .gitignore care va ignora:

  • Toate fișierele cu extensia .log
  • Directorul node_modules/

Am adăugat fișierul și am făcut commit cu mesajul corespunzător.

II. Lucrul cu Branch-uri

Creare branch Testare-1 1/3
git checkout -b Testare-1

Am creat un nou branch numit Testare-1 și am trecut pe acesta.

Acest branch va fi folosit pentru dezvoltarea structurii proiectului web.

Creare structură proiect web 2/3
# Creare directoare și fișiere mkdir css js image config touch css/style.css js/script.js image/placeholder.txt config/config.json touch index.html portfolio.html # Commit-uri pentru fiecare componentă git add css/ && git commit -m "Creat director css și fișierul style.css" git add js/ && git commit -m "Creat director js și fișierul script.js" git add image/ && git commit -m "Creat director image și fișierul placeholder.txt" git add config/ && git commit -m "Creat director config și fișierul config.json" git add index.html && git commit -m "Creat fișierul index.html cu referinte" git add portfolio.html && git commit -m "Creat fișierul portfolio.html cu referinte"

Am creat structura completă a proiectului web:

  • Directorul css/ cu fișierul style.css CSS
  • Directorul js/ cu fișierul script.js JavaScript
  • Directorul image/ cu fișierul placeholder.txt TXT
  • Directorul config/ cu fișierul config.json JSON
  • Fișierul index.html HTML
  • Fișierul portfolio.html HTML

Am făcut commit-uri separate pentru fiecare componentă adăugată.

Push branch Testare-1 pe GitHub 3/3
git push origin Testare-1

Am încărcat branch-ul Testare-1 pe repository-ul remote de pe GitHub.

GitHub sugerează crearea unui pull request pentru acest branch.

Branch încărcat cu succes pe GitHub

III. Operațiuni de Merge

Merge branch Testare-1 în main 1/2
git checkout main git merge Testare-1 git push origin main

Am efectuat merge-ul branch-ului Testare-1 în branch-ul principal main.

Operațiunea a fost un fast-forward merge, deoarece branch-ul main nu avea commit-uri noi față de Testare-1.

Am încărcat modificările pe GitHub după merge.

Starea finală a proiectului 2/2

Structura finală a proiectului după merge:

Branch-uri main și Testare-1

Directoare css/, js/, image/, config/
